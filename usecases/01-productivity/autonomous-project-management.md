# 自主项目管理 (Autonomous Project Management)

## 案例概述

这是一个去中心化的项目管理模式，子代理通过共享 `STATE.yaml` 文件自主工作，而不是依赖中央协调器。

## 解决的问题

传统协调器模式会造成瓶颈——主代理被协调任务压垮，对复杂项目（如多仓库重构或内容流水线）效率低下。

## 工作原理

1. 主代理接收任务并生成具有特定范围的子代理
2. 子代理读取 `STATE.yaml` 找到分配的任务
3. 子代理自主工作并更新 `STATE.yaml` 进度
4. 其他代理轮询 `STATE.yaml` 拾取未阻塞的工作
5. 主代理定期检查状态并调整优先级

## 核心技术：STATE.yaml

`STATE.yaml` 文件作为项目协调的真实来源，跟踪任务的状态、所有者、时间戳和依赖项。这种基于文件的方法比消息传递更具可扩展性，因为代理可以并行工作而无需持续的通信开销。

## 关键要求

- `sessions_spawn` / `sessions_send` 用于子代理管理
- 文件系统访问用于读写 STATE.yaml
- Git 用于状态版本控制（可选但推荐）

## 关键洞察

- **"STATE.yaml > 协调器"**：基于文件的协调比消息传递更具可扩展性
- 主代理做得越少，响应越快——这是"CEO 模式"，主会话只处理战略
- 使用标签约定如 `pm-{project}-{scope}` 便于跟踪
- 提交 STATE.yaml 更改到 git 以获得完整的审计日志

## 关联技能

| 技能 | 用途 |
|------|------|
| subagent-driven-development | 子代理调度开发 |
| writing-plans | 编写项目计划 |
| executing-plans | 执行计划 |
| verification-before-completion | 完成前验证 |

## 使用场景

- 多仓库代码重构
- 内容创作流水线
- 大型文档编写项目
- 多模块应用开发
